# Voting and Aggregation

Private Council does not use majority vote as the final decision rule.

## Voting Is a Signal, Not a Verdict

If five roles prefer one option and one role objects, the majority is useful information, but it is not a decision.

The minority objection may identify:

- a hidden risk
- a false assumption
- an execution blocker
- a value conflict
- missing evidence

## What the System Should Aggregate

The system should aggregate:

- claims
- assumptions
- risks
- evidence
- objections
- confidence
- predictions
- role preferences

It should not collapse them into a single automatic verdict.

## Cross-validation Over Majority Rule

Important conclusions should be checked through independent roles or models.

The system asks:

- Did multiple roles identify the same risk?
- Did different models reach the same assumption independently?
- Is there a minority objection with high impact?
- Does the recommendation depend on one fragile premise?
- What evidence would change the recommendation?

## Human Override Is Expected

If the council recommends one path and the user chooses another, that is not a product failure.

The system records:

- what the council recommended
- what the user chose
- why the user modified or rejected the recommendation
- what should be reviewed later
