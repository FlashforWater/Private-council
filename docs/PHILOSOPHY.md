# Design Philosophy

Private Council is built around one product claim:

> A person should be able to borrow the thinking quality of a mature team without surrendering the final decision.

## Council Advises. Human Decides.

The council can clarify, challenge, synthesize, and recommend, but it does not make the final decision.

The system should always bring important conclusions back to the user:

- Do you accept this recommendation?
- Do you want to modify it?
- Do you need more information?
- Do you want to defer?
- What next action are you willing to commit to?

## Preserve Disagreement Before Synthesis

Disagreement is not noise. It is often the highest-value part of deliberation.

Private Council should show:

- who supports which option
- who challenges it
- which assumptions drive the split
- which risks are shared
- which minority objection might overturn the recommendation

## Decision Canvas Over Chat

Chat is an input surface. The Decision Canvas is the product artifact.

The canvas records:

- problem frame
- options
- criteria
- claims
- assumptions
- risks
- evidence
- disagreements
- recommendation
- human decision
- next action
- review plan
- predictions

## Roles, Models, Opinions, and Decisions Are Separate

Private Council separates four things:

- Role: a stable cognitive responsibility
- Model: the temporary engine behind a role
- Opinion: a structured contribution in one session
- Decision: the user's final judgment

This avoids turning model brands into product personalities and allows routing to evolve.

## Every Decision Should Become Learning Material

A decision should not end when advice is generated.

Private Council records predictions and review plans so the user can later ask:

- What actually happened?
- Which assumptions were wrong?
- Which risks mattered?
- Which role or model was useful?
- What should I do differently next time?
